## Problem

Amazon handles over 5 million deliveries every day. Each driver covers a neighborhood with a dozen or more stops, and the order in which those stops are visited matters far more than it might initially seem. With 15 stops alone, there are more than 87 billion possible route orderings, making brute-force exploration impractical.

This project models the problem as a Travelling Salesman Problem and solves it using the Held–Karp dynamic programming algorithm. Instead of checking every permutation, the algorithm breaks the problem into smaller reusable subproblems to compute the exact optimal route efficiently.

To compare performance, the project also includes several baseline and heuristic approaches: random ordering, greedy nearest-neighbor, 2-opt local search, and Or-opt for larger-scale optimization scenarios.

The project includes:

* a 100-stop dataset covering real delivery zones in the College Park and DC metro area,
* an interactive web demo,
* a CLI-based evaluation runner,
* and a complete evaluation suite with six generated result plots.

## Setup

**Requirements:** Python 3.8+, `matplotlib`, and `numpy`. No additional dependencies are needed for the core algorithms.

```bash
git clone https://github.com/AmeyHengle/MSML606_DynamicP.git
cd MSML606_DynamicP
pip install matplotlib numpy
```

### Run the CLI demo

Run the 15-stop sample route:

```bash
python main.py
```

With additional options:

```bash
python main.py --stops 12 --seed 42    # reproducible random route
python main.py --stops 20 --no-hk      # skip Held–Karp for larger n
```

### Generate the 100-stop dataset

The dataset is already included, but it can be regenerated if needed:

```bash
python generate_dataset.py
```

### Run the full evaluation suite

This generates all plots and a CSV file inside `results/`:

```bash
python evaluate.py          # full run, takes 2 to 3 minutes
python evaluate.py --fast   # quick version, roughly 30 seconds
```

### Open the web demo

No server setup is required. Open the file directly in a browser:

```bash
open app/index.html         # macOS
xdg-open app/index.html     # Linux
```

The demo contains four sections:

* an animated route comparison,
* a live complexity simulation,
* eight geometric edge-case scenarios,
* and a step-by-step walkthrough of the Held–Karp algorithm.

## Results

The algorithms were evaluated on datasets ranging from N = 5 to 50 stops, averaging 5 trials per value of N using samples from the 100-stop College Park dataset.

### Mean total route distance (lower is better)

| N  | Random | Greedy | 2-opt | Or-opt | Held–Karp (exact) |
| -- | ------ | ------ | ----- | ------ | ----------------- |
| 5  | 0.319  | 0.267  | 0.260 | 0.260  | 0.260             |
| 10 | 0.633  | 0.369  | 0.340 | 0.339  | 0.339             |
| 15 | 0.906  | 0.465  | 0.393 | 0.390  | 0.390             |
| 20 | 1.363  | 0.526  | 0.436 | 0.434  | —                 |
| 30 | 1.883  | 0.579  | 0.495 | 0.487  | —                 |
| 50 | 3.158  | 0.711  | 0.572 | 0.558  | —                 |

At N = 15, Or-opt matches the Held–Karp solution exactly, achieving an approximation ratio of 1.000. At N = 50, Or-opt reduces total route distance by 21.5% compared to the greedy baseline.

Held–Karp guarantees the optimal solution, but its exponential time complexity of O(2^n \cdot n^2) limits practical usage to roughly N ≤ 15.

### Mean runtime (milliseconds)

| N  | Greedy | 2-opt | Or-opt | Held–Karp  |
| -- | ------ | ----- | ------ | ---------- |
| 10 | 0.04   | 0.06  | 0.10   | 6.0        |
| 15 | 0.10   | 0.16  | 0.23   | 499.9      |
| 50 | 0.83   | 1.69  | 4.04   | infeasible |

### Evaluation plots

Generated in `results/` by `evaluate.py`:

* `1_distance_vs_n.png` — route distance versus stop count across all algorithms
* `2_runtime_vs_n.png` — runtime on a logarithmic scale, highlighting the Held–Karp infeasibility cliff
* `3_approximation_ratio.png` — how closely each heuristic approaches the exact optimal solution (N ≤ 15)
* `4_improvement_over_greedy.png` — percentage distance reduction relative to greedy routing
* `5_trial_variance.png` — box plots showing consistency across randomized trials
* `6_route_map_n20.png` — geographic route comparisons for the three heuristic methods at N = 20
